Syntax-Aenderungen

  • $var statt !var
  • = ist tot
  • neue String-Interpretation
  • - in Namen (Mixins, Variablen)

$var

style.sass 

$height: 34px

h1
  height: $height
  line-height: $height/2 + 2px
style.css 
h1 {
  height: 34px;
  line-height: 19px; }

Zum Vergleich

style.sass2 
!height= 34px

h1
  height= !height
  line-height= !height/2 + 2px
  font-weight: bold
style.sass3 
$height: 34px

h1
  height: $height
  line-height: $height/2 + 2px
  font-weight: bold

SCSS - Um es mal erwaehnt zu haben

Nur eine andere Syntax mit den selben Features. Soll es Umsteigern vereinfachen.

$height: 34px;

h1 {
  height: $height;
  line-height: $height / 2 + 2px; }
  • Curly Braces statt Python-Like Einrueckung
  • @mixin fuer Mixin-Definitionen
  • @include fuer Mixin-Aufrufe
  • Wird Sass niemals ersetzen!

@extend - Selector Inheritance

error.html 
<div class="error seriousError">
Oh no! You've been hacked!
</div>
style.css 
.error {
  border: 1px #f00;
  background-color: #fdd;
}
.seriousError {
  border-width: 3px;
}

Nervig

Vergisst man .error, sieht es scheisse aus.

Besser: @extend

style.sass 
.error
  border: 1px #ff0
  background-color: #fdd

.seriousError
  @extend .error
  border-width: 3px
style.css 
.error, .seriousError {
  border: 1px yellow;
  background-color: #ffdddd; }

.seriousError {
  border-width: 3px; }

Cascading Stylesheets

Eine unbequeme Wahrheit

.angewendet links nach rechts wird CSS

                    #main-content ul.side-menu li div

Folgen

  • Es wird immer zunaechst das CSS analysiert.
  • Je unspezifischer die Selektoren, besonders die letzten, desto langsamer das Rendering.
  • miGadgets ist unglaublich langsam! (screen.css ~= 7300 Zeilen CSS) Firebug beinahe unnutzbar.

Mehr Folgen

  • Gerade im Hinblick auf mobile Geraete, kann man sich Latenzen durch CSS nicht erlauben.
  • Gewisse DOM-Operationen erfordern ein Rerendering der gesamten Seite.

Lektionen

  • CSS klein halten.
  • CSS dynamisch nachladen!
  • Keine Monster-Stylesheets
